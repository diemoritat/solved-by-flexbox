Introdução

O CSS tem sofrido com a falta de mecanismos de layout por tempo demais. Transições, animações, filtros, todos são adições ótimas e úteis à linguagem, mas eles não tratam dos maiores problemas que desenvolvedores vem reclamando pelo que parece ser uma eternidade.

Finalmente, graças ao Flexbox, temos uma solução.

Este site não é mais um framework CSS. Em vez disso, o propósito desse site é mostrar problemas que antes eram difíceis ou impossíveis de serem resolvidos apenas com CSS, agora sendo triviais com Flexbox. E com o lançamento recente do Internet Explorer 11 e Safari 6.1, a última especificação do Flexbox é suportada em todo browser moderno.

Confira os demos abaixo. Veja os estilos no Web Inspector ou entre no código fonte para ver o quão fácil o layout com CSS será quando Flexbox virar popular.

Ressalvas e Bugs Conhecidos

  • IE 10 tem suporte ao Flexbox, mas para a versão de rascunho da especificação atual:(display:flexbox).
  • Safari 6 e versões mais novas oferecem suporte à versão original da sintaxe do Flexbox, que agora é obsoleta: (display:box).
  • Firefox 27 e versões mais novas não oferecem suporte a flexboxes multilinha. Veja esse bug para mais detalhes.
  • Para uma comparação completa de suporte oferecido por browsers, confira caniuse.com/flexbox

Sobre o Código

Todos os exemplos de código nesse site mostram como resolver um problema em particular usando Flexbox. Eles mostram apenas o código que é necessário para fazer as demos funcionarem em um browser que ofereça suporte completo à especificação atual. Alguns browsers não oferecem suporte total com a última especificação, então infelizmente, algumas soluções alternativas foram necessárias.

Soluções para browsers sem suporte completo não são mostradas nos exemplos de código, mas se você estiver curioso sobre esses detalhes de implementação, você pode verificar o código fonte. Cada demo possui link para sua fonte, e todas as alternativas específicas para esses browsers estão bem documentadas, então não tenha medo de dar uma olhada.

Os prefixos e traduções das propriedades flex atuais para as suas equivalentes antigas são gerenciadas pelo autoprefixer. Se você está escrevendo código com Flexbox e não está usando autoprefixer, bom, você está cometendo um engano terrível.

A convenção utilizada para nomenclatura de classes nos exemplos de código foi retirada do SUIT CSS, que é baseado na metodologia BEM. Cada exemplo inclui um ou mais componentes CSS reutilizáveis, permitindo que você adapte ou copie esses padrões para seus projetos. Links são providenciados para seus respectivos componentes em cada página de exemplo.

Se você encontrar um erro ou quiser sugerir algum exemplo adicional, sinta-se livre para abrir uma issue ou um pull request em Github.
